# Contributing to Parevo Flow

First off, thank you for considering contributing to Parevo Flow! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
Before creating a bug report, please check that an issue doesn't already exist. When filing a bug, please include:
- A clear and descriptive title
- Steps to reproduce the behavior
- Expected vs Actual behavior
- Go version and OS

### Suggesting Enhancements
We love new ideas! If you have a suggestion for an enhancement, please open an issue with the "feature request" template.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`go test ./...`).
4. Make sure your code is linted.
5. Issue the Pull Request!

## Development Setup
1. Clone the repo.
2. Run `docker-compose up -d` to start databases.
3. Run tests using `go test ./...`.

---
Happy coding!
