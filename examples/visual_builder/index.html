🧪 Testing Guide - Employee Onboarding Workflow

📋 Workflow Overview

Bu workflow tüm node tiplerini test eder:

  • db_query - Kullanıcı bilgisi çeker
  • db_write - Audit log yazar & status günceller
  • log - Console mesajları
  • wait - 3 saniye bekler
  • signal - Manuel tetikleme bekler (3 adet)
  • http - Dış API çağrısı
  • ai - 🤖 AI analiz (ChatGPT/Claude/Gemini)
  • progress - İlerleme gösterir
  • condition - Doğrulama dallanması

🚀 Nasıl Test Edilir?

1. Deploy & Run butonuna tıklayın

→ Workflow başlar, user_id=2 (Zeynep Yılmaz) ile çalışır

2. İlk Signal Node (Belge Yükleme - Boş Geçilir)

  • Sarı animasyonlu node görünür
  • Control panel'den node'u seçin
  • Key/Value boş bırakın (sadece devam etmek için)
  • ⚡ Trigger Signal tıklayın → İş akışı devam eder

3. HTTP API Çağrısı

→ Otomatik jsonplaceholder'a GET request atar (beklemez)

4. ⚠️ KRİTİK: Admin Onay Signal

ÖNEMLİ: Bu signal'dan sonra Condition node geliyor! Burada gönderdiğiniz status değeri Condition tarafından kontrol edilecek.

  • Sarı node bekler, dropdown'dan seçin
  • Key: status yazın (tam olarak)
  • Value: verified (başarı için) VEYA başka bir değer (hata için)
  • ⚡ Trigger Signal → Bu veri sonraki Condition node'a geçer!

🔀 Condition Dallanması (Signal Sonrası)

Eğer Signal'da status: verified gönderirseniz:

  • ✅ Yeşil dal (TRUE) → Eğitim başlar
  • DB'de user status verified olur
  • 3. Signal node (Eğitim tamamlama) gelir
  • Final: 100% progress

Eğer başka değer gönderirseniz (örn: rejected veya Key/Value boş):

  • ❌ Kırmızı dal (FALSE) → Hesap askıya alınır
  • DB'de user status suspended olur
  • Progress 0% olur
  • Not: Condition sadece status == "verified" kontrol eder

🎨 Visual Feedback

  • Yeşil border: COMPLETED
  • Sarı animasyon: WAITING (Signal bekliyor)
  • Kırmızı border: FAILED

🗄️ Database Inspection

Console'da SQL logları görürsünüz. İsterseniz:

http://localhost:8080/api/db?table=users

http://localhost:8080/api/db?table=audit_log

💡 Pro Tips

  • Signal → Condition İlişkisi: Signal'da girdiğiniz Key/Value'lar sonraki node'lara taşınır!
  • AI Node: API key değiştirin veya node'u silin. OpenAI/Claude/Gemini desteklenir!
  • Farklı user_id test etmek için URL'yi değiştirin: /api/run?user_id=1
  • Console loglarını açın (F12) → SQL sorguları ve node çıktılarını görürsünüz
  • Node'ları drag & drop ile ekleyebilirsiniz
  • Her node'a tıklayarak config'ini görebilirsiniz
  • Reset button ile baştan başlayabilirsiniz
  • Hata yaparsanız: Sayfayı yenileyin ve tekrar başlayın
